# Git Lesson

## リモートリポジトリとローカルリポジトリとはそれぞれ何でしょうか？
### ローカルリポジトリは、自分のPC上にある保管場所で、個人の開発作業に使用される。
### リモートリポジトリは、ネットワーク上にある保管場所で、複数の開発者とコードを共有するために使用される。


## プッシュとマージの違いは何でしょうか？
### プッシュは、ローカルリポジトリの編集内容をリモートに送る操作。　
### マージは、別のブランチの変更内容をブランチに取り込む操作。

## コミットとプッシュの違い
### コミットは、 ローカルリポジトリに編集内容を記録する操作。
### プッシュは、ローカルリポジトリのコミットをリモートリポジトリに送り共有する操作。


## コミットのメッセージはどのように書いてあげるのが最適でしょうか？
### 変更内容が一目で分かるように書く。


## ローカルでマージするフローと、プルリクエストでマージするフローの違いは何でしょうか？
### 前者はコードレビューをする前に反映されるため、コードにバグがあっても気づかない可能性があるが、後者の場合はコードレビューをしてからマージするため事前にバグを発見できる。


## コンフリクトを起こしてしまった場合、どう対処すべきですか？
### 先にマージされた変更内容、後にマージしようとしている変更内容のどちらか、またはどちらの変更内容も取り込む。
### 少しでも意図が読み取れない処理とぶつかってしまった場合は、そのソースコードを書いた人と相談しながら作業を進める。
